# Assume:
``` 
- `main` = production
- `develop` = active development
- PR-based workflow
- Feature branches required
```

## ==> Scenario 1 — Project Setup (Day 1 Task)

You joined the company.

=> Task:

1. Initialize repository
2. Create `main`
3. Create `develop` from `main`
4. Add README and initial commit
5. Push both branches to remote

Goal:
Understand branch structure and remote setup.

## ==> Scenario 2 — First Feature Development

You are assigned:

> Build Login API

=> Rules:

- Must branch from `develop`
- Branch name format: `feature/login-api`
- Minimum 3 commits
- Push branch to remote
- Do NOT merge directly

Goal:
Understand feature branch workflow.
